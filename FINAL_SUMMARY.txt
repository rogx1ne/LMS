PROBLEM STATEMENT
════════════════════════════════════════════════════════════════════════════

After running the LMS Setup Wizard, critical functionality was broken:
  ❌ Users could not issue books
  ❌ Users could not place orders
  ❌ Fresh installations still contained old dummy data
  ❌ Windows Oracle 11g detection was failing
  ❌ Installation location path was not being captured

The application worked BEFORE setup, but broke AFTER setup completed.
This indicated a database initialization or configuration problem.


ROOT CAUSE ANALYSIS
════════════════════════════════════════════════════════════════════════════

Issue #1: Location Path Not Captured
   Root Cause: Setup form field was local variable, not class member
   Impact: Installation location not passed to database
   Fix: Added locationPathField as JTextField class member (Line 50)

Issue #2: Oracle 11g Not Detected
   Root Cause: Hardcoded single connection string (localhost:1521:xe)
   Impact: Windows 11g, alternate ports, Podman not supported
   Fix: Try 5 different configs (Lines 559-600)

Issue #3: script.sql Path Resolution Failing
   Root Cause: Single hardcoded path, production uses different locations
   Impact: Setup fails in standalone/production scenarios
   Fix: Search 3 locations: current dir, JAR dir, project root (Lines 394-413)

Issue #4: No Post-Installation Options
   Root Cause: Installation just ended, no way to create shortcuts
   Impact: User had to manually configure after setup
   Fix: Added 3 action buttons on completion page (Lines 441-492)

Issue #5: Fresh Install Still Has Old Data
   Root Cause: script.sql only deleted 2 tables, not all 13
   Impact: Old books, orders, students, sellers still visible
   Fix: Comprehensive DELETE for all 13 tables in proper FK order (Lines 271-294)
