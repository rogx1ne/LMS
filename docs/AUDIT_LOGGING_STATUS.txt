KEY IMPLEMENTATION DETAILS
================================================================================

PATTERN USED: Transaction-Safe Audit Logging
- Each operation manages its own database connection
- AutoCommit is explicitly disabled
- Operation and audit log are committed/rolled back together
- No orphaned audit entries on errors
- User ID (performedBy) is captured from CurrentUserContext
- Descriptive messages include relevant IDs and details

EXAMPLE FLOW:
1. User performs action (e.g., issue book) in UI
2. CirculationController.issueBook() is invoked
3. circularDAO.issueBook(..., CurrentUserContext.getUserId()) is called
4. In DAO: setAutoCommit(false)
5. Issue transaction is executed (INSERT into TBL_COPY_TRANSACTION)
6. AuditLogger.logAction(conn, "USER-001", "Circulation", "Issued book...") 
7. Transaction committed (both operation and audit log)
8. If error: both operation and audit log rolled back

TRANSACTION SAFETY GUARANTEE:
- If operation succeeds: Audit log is recorded
- If operation fails: Audit log is NOT recorded (no orphaned entries)
- If audit log fails: Entire transaction rolled back (data consistency)
- Referential integrity maintained at all times
